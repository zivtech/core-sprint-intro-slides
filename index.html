Core Contributions

Why Do They Matter to Zivtech?

Exposure

  • The Drupal Marketplace algorithm uses core contributions to rank companies on their involvement. This is one of the reasons Acquia at the top of the list.
  • The more we are active in core issue queues the more Zivtech's name is seen as a house brand in the community.
  • Some companies look for Drupal shops that are active in core development, as they know those shops are competent in what they do.

Experience

  • Working on high level issues gives us a better understanding of what issues we might face on projects and get them resolved easier when they are an issue on a client project.
  • Getting involved in core issues allows us to learn more about Drupal core than we would just building sites.
  • Being involved in issues gives us more knowledge of Drupal's core APIs and better ways to build sites through community conversations.

How Do We Get Involved?

Pick a Niche and Scratch It!

I picked up some cards during the Drupal Core sprint in Baltimore which gave some great info for some specific roles that are involved in a sprint.

Explorer

You are familiar with Drupal and ready to start exploring Drupal 8.

How Can I Help?

  • Start exploring the Drupal Core issue queue to look for issues in your skill set.
  • Get a Drupal 8 local installation setup for testing and debugging issues.
  • Focus on issues that are higher priority.

Documenter

You are a Community Contributor who can write well in English, and are ready to share knowledge about Drupal 8 with others.

How Can I Help?

  • Join #drupal-docs on IRC.
  • Review the documentation style guide, and instructions for taking screenshots, and uploading images.
  • Figure out how to take screenshots on your OS.
  • Read the current documentation priorities.
  • Work on writing, editing, or reviewing the Drupal 8 User Manual.
  • Fix documentation page titles.
  • Add keywords to a documentation page.
  • Incorporate the content of a comment into the documentation itself.

Issue Mover

You are a Community Contributor and ready to start moving Drupal 8 issues forward.

How Can I Help?

  • Install Drupal 8 locally or use simplytest.me.
  • Install Dreditor.
  • Figure out how to take screenshots on your OS.
  • Find a Needs issue summary update issue and if it is missing steps to reproduce, write them. Bonus points if you include relevant screenshots.
  • Triage an issue without any replies.
  • Test an issue tagged Needs manual testing.
  • Find a Novice issue with a task you can perform, do it, and have someone check your post.

Developer

You are an Issue Mover or Documenter and ready to dig into some Drupal 8 code.

How Can I Help?

  • Install a local *AMP stack (e.g., Dev Desktop).
  • Get an IDE such as PHPStorm or Sublime Text.
  • Set up git locally.
  • Git clone Drupal 8 and install locally.
  • Run a simpletest locally.
  • Run phpunit tests locally.
  • Find a Needs reroll issue and re-roll a patch that no longer applies.
  • Find a Needs tests issue and write a unit test.
  • On a problem you have encountered, upload a partially working patch (partially done tests counts).

Mentor

You are an Issue Mover, Documenter or Developer and are ready to help others learn to contribute too.

How Can I Help?

  • Do some issue queue gardening.
  • Review Novice and Meta issues so you have tasks to match new contributors with.
  • Wear comfortable shoes and a smile.
  • Drink plenty of water.
  • Ask and answer questions in public, and expose your reasoning.
  • Ask for help when you need it.
  • Review.
  • Review, review, review. Sprinters need feedback!

The End.
